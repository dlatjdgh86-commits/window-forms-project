## 2. 크롤링 우선순위

```
1순위 (MVP 필수)
  ├── 광운대 공식 사이트 - 학사일정     ← 로그인 불필요, 안정적
  └── 광운대 공식 사이트 - 학사공지     ← 로그인 불필요, 안정적

2순위 (로그인 구현 후)
  ├── LMS - 과제 마감일                ← 개인화된 TODO 핵심 데이터
  └── KLAS - 수강 정보                 ← 수업 기반 일정 생성

3순위 (추후 검토)
  └── 에브리타임                       ← 이용약관 검토 필요
```

## 3. 데이터 수집 방법

### 3-1. 공통 크롤링 스택

```python
# 필요 라이브러리
requests==2.31.0       # HTTP 요청
beautifulsoup4==4.12.0 # HTML 파싱
lxml==4.9.3            # HTML 파서 (bs4 백엔드)
selenium==4.15.0       # JS 렌더링 필요 시 (선택)
```

### 3-2. 학사일정 파싱 예시 구조

```python
# 수집할 데이터 필드
{
  "title": "2025학년도 1학기 중간고사",
  "start_date": "2025-04-14",
  "end_date": "2025-04-18",
  "category": "시험",          # 시험 / 수강신청 / 방학 / 행사 / 기타
  "source": "kw.ac.kr",
  "priority": "high",          # LLM이 분석하여 부여
  "raw_text": "원본 텍스트"
}
```

### 3-3. TODO 생성 데이터 필드

```python
# LLM이 생성할 TODO 구조
{
  "id": "uuid",
  "title": "중간고사 준비 - 자료구조",
  "due_date": "2025-04-14",
  "priority": "high",           # high / medium / low
  "category": "학업",           # 학업 / 행정 / 장학 / 기타
  "source_event": "중간고사",   # 근거가 된 학사일정
  "is_done": false,
  "created_at": "2025-04-01T09:00:00"
}
```

## 4. 크롤링 제한 및 대응 방안

| 문제 | 대응 방안 |
|------|-----------|
| IP 차단 | 요청 간 1~2초 딜레이, 재시도 로직 구현 |
| HTML 구조 변경 | 크롤러 모듈화, 파싱 로직 분리하여 유지보수 용이하게 |
| 로그인 세션 만료 | 세션 재발급 자동화, 에러 핸들링 |
| 서버 점검 | 마지막 성공 데이터 로컬 캐싱 (SQLite) |
| HTTPS 인증서 오류 | `verify=False` 옵션 (개발 환경 한정) |
